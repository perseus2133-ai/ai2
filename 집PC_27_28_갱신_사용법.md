# 집 PC 27/28 컨센서스 갱신 — 사용법 (자가적응 v2)

## 왜 필요한가

- **2027E·2028E 연간 컨센서스는 FnGuide에만** 있다.
  (네이버/wisereport = 26E까지, 퀀트킹 CSV export = 26년1Q(E)까지 — 전수 확인됨)
- FnGuide는 지역 차단이 아니라 **누적 요청 기반 소프트 봇차단**이다:
  한 IP에서 요청이 쌓이면 gicode를 무시하고 삼성전자 기본 페이지를 돌려준다.
  requests든 실제 브라우저든 동일 → 도구가 아니라 **IP 상태**의 문제.
- 한번 차단되면 쿨다운이 길다(시간~하루). 그래서 이 스크립트는
  **차단에 부딪히기 전에 스스로 멈춘다.**

## 역할 분담

| 주체 | 하는 일 | 빈도 |
|---|---|---|
| GitHub Actions (자동) | 네이버: 현재가·시총·26E·수급·기술적 | 매일 새벽 |
| **집 PC (이 스크립트)** | FnGuide 27/28E fresh | **매일 1~2회 더블클릭** |
| carry-forward (자동) | 공백을 최근 값으로 유지 | 크롤 때마다 |

## 사용법

**`27_28_갱신.bat` 더블클릭.** 끝. (시작 시 git pull → 수집 → CSV 보강 → push 자동)

## 자가 적응이 하는 일

- **안전 한도 학습**: `.fnguide_stats.json`(이 PC 전용, git 미공유)에
  이 IP의 안전 요청 수를 기록한다.
  - 첫 실행: 120개 요청 후 자발적 중단 (보수적 시작)
  - 차단 당하면: 차단 지점의 70%로 한도 하향
  - 무차단 완주하면: 한도 15% 상향 (최대 800)
- **한도 도달 → 자발적 중단**: 차단을 아예 안 당하니 IP가 깨끗하게 유지되어
  **같은 날 몇 시간 뒤 또 실행 가능**하다.
- **7일 순환**: 최근 7일 내 fresh 받은 종목은 건너뛴다. 매일 돌리면
  전체 종목이 자동으로 순환 갱신된다.
- **시작부터 차단이면**: 요청 1개만 보내고 즉시 종료 (쿨다운 연장 방지).
  몇 시간 후 다시 실행.

## 메시지 읽는 법

| 메시지 | 의미 | 할 일 |
|---|---|---|
| `🛑 요청 한도 N개 도달 → 자발적 중단` | 정상. IP 깨끗함 | 몇 시간 뒤 또 실행하면 이어받음 |
| `⛔ 시작부터 차단 상태` | 이 IP 쿨다운 중 | 몇 시간~다음날 다시 |
| `⛔ 연속 3회 차단 → 한도 도달` | 한도 초과 (다음부턴 더 일찍 멈춤) | 다음날 다시 |
| `✅ git push 완료` | 갱신분이 GitHub 반영됨 | 없음 |

## 옵션

| 명령 | 효과 |
|---|---|
| `27_28_갱신.bat` | 기본 (자동 학습 한도) |

| `python refresh_27_28.py --no-push` | git pull/push 없이 (테스트) |

## 자동화 (선택) — Windows 작업 스케줄러

1. 시작 → "작업 스케줄러" → 기본 작업 만들기
2. 트리거: 매일, PC 켜져 있는 시간 (예: 오전 9시 / 오후 7시 두 개 등록 가능)
3. 동작: 프로그램 시작 → `27_28_갱신.bat`
