# 집 PC 27/28 컨센서스 갱신 — 사용법

## 왜 필요한가

- **2027E·2028E 영업이익/매출 컨센서스는 FnGuide(comp.fnguide.com)에만** 존재한다.
  (네이버·wisereport는 2026E까지가 한계)
- FnGuide는 **GitHub Actions의 미국 Azure IP를 지역 차단**한다 (모든 요청에
  '삼성전자 고정 페이지' 반환). → 클라우드 자동 크롤로는 27/28을 못 받는다.
- **한국 IP(집 PC)에서만** 27/28을 받을 수 있다.

그래서 역할을 나눈다:

| 주체 | 하는 일 | 빈도 |
|---|---|---|
| GitHub Actions (자동) | 네이버 데이터: 현재가·시총·거래량·26E·수급·기술적 | 매일 새벽 |
| **집 PC (이 스크립트)** | FnGuide 27/28E만 보충 | 주 1~2회 (또는 매일) |
| carry-forward (자동) | 그 사이 공백을 최근 값으로 유지 | 크롤 때마다 |

## 사용법 (가장 쉬움)

**`27_28_갱신.bat` 더블클릭.** 끝.

- 자동으로: FnGuide에서 27/28 받기 → CSV 보강 → git push
- 받는 데는 몇 분~십수 분. 진행 상황이 창에 표시된다.
- 끝나면 아무 키나 눌러 창을 닫는다.

## FnGuide 차단(한도)에 대해

FnGuide는 한 IP에서 **짧은 시간에 많이 요청하면 차단**하고, 한 번 차단되면
**쿨다운이 길다(시간~하루)**. 그래서 이 스크립트는:

- 요청을 **천천히**(기본 1.5초 간격) 직렬로 보낸다.
- **차단(삼성 고정 페이지)을 감지하면 즉시 멈춘다** (연속 4회). 더 때려서
  쿨다운을 늘리지 않는다.
- **가장 오래된(stale) 종목부터** 갱신하고, 오늘 이미 받은 건 건너뛴다(resume).
  → 차단으로 중간에 멈춰도, **다음에 다시 돌리면 이어서** 받는다.

즉 한 번에 다 못 받아도 괜찮다. 며칠에 걸쳐 돌리면 669개가 순환 갱신되고,
그 사이는 carry-forward가 값을 유지한다. (컨센서스는 월 단위로 천천히 변하므로 충분)

### 차단이 잦으면
- 간격을 늘려서 실행: `python refresh_27_28.py --interval 3.0`
- 또는 하루 뒤(쿨다운 후) 다시 `27_28_갱신.bat` 더블클릭

## 자동화 (선택) — Windows 작업 스케줄러

매번 더블클릭이 번거로우면, PC가 켜져 있는 시간에 자동 실행되게 등록:

1. 시작 → "작업 스케줄러" 실행
2. 우측 "기본 작업 만들기"
3. 트리거: 매일 (PC를 주로 켜두는 시간, 예: 오전 9시)
4. 동작: "프로그램 시작" → 찾아보기 → `27_28_갱신.bat` 선택
5. 완료. PC가 그 시간에 켜져 있으면 자동 실행된다. (꺼져 있으면 그날 skip)

## 옵션

| 명령 | 효과 |
|---|---|
| `27_28_갱신.bat` 더블클릭 | 기본 실행 (간격 1.5초) + push |
| `python refresh_27_28.py --interval 3.0` | 간격 3초 (차단 잦을 때) |
| `python refresh_27_28.py --no-push` | git push 없이 로컬만 (테스트용) |
